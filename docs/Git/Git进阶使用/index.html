二、Git进阶使用

Git分支​

主干分支​

默认情况下，Git软件就存在分支的概念，而且就是一个分支，称之为master分支，也称之为主干分支。这就意味着，所有文件的版本管理操作都是在master这一个分支路线上进行完成的。

其他分支​

就像之前说的，如果仅仅是一个分支，在某些情况并不能满足实际的需求，那么就需要创建多个不同的分支。

  • 创建分支
# git branch 分支名称
git branch b1
git branch b2

我们现在创建了两个分支，不过这两个分支都是master主干分支为基础的。

  • 查看分支
git branch -v
  • 切换分支

我们将工作线路切换到b1

# git switch 分支名称
git switch b1
#创建并切换分支
git switch -c test

如果此时我们在b1分支提交b1.txt文件，查看分支状态会发现不同分支版本进度信息发生了改变，但是我们切换回主干分支后，b1.txt文件就不存在了，因为对应的版本信息不一样。

  • 删除分支

如果觉得某一个分支建立的不太理想或已经没有必要在使用了，那么我们是可以将这个分支删除的。

#git branch -d 分支名称
git branch -d b2
  • 合并分支

无论我们创建多少个分支，都是因为我们需要在不同的工作环境中进行工作，但是，最后都应该将所有的分支合在一起，形成一个整体，作为项目的最终结果。

首先我们在主分支master上面创建master.txt文件，然后我们在主分支上面创建新的b1分支，切换到b1分支创建b1.txt文件。

这时主分支上面仅有master.txt文件，我们将b1分支合并到master主分支上面来，首先我们切换回主干分支，然后执行分支合并指令

git merge 分支名称
git merge b1

这时我们在主分支就可以看到两个文件了，master.txt、b1.txt,分支合并成

在开发中除了通过merge来合并分支外，还可以通过变基来完成分支的合并，git rebase master，变基就是改变基底。 我们通过merge合并分支时，在提交记录中会将所有的分支创建和分支合并的过程全部都显示出来，这样当项目的开发过程复杂，过程波折的时候，需要反复的创建、合并、删除分支。这样一来代码的提交记录会十分混乱。 原理： 1.当我们发起变基时，git首先找到两条分支的最近的共同祖先 2.然后对比当前分支相对于祖先的历史提交，并且讲它们提取出来存储到一个临时文件中 3.将当前部分分支指向目标分支的基底 4.以当前基底开始，重新执行历史操作 变基和merge对于合并分支来说最终的结果是一样的！但是变基会使得代码的提交记录更加简洁、清晰！在大部分情况下，合并和变基是可以互换的，但是如果分支已经提交给了远程仓库，那么这时尽量不要变基。

Git冲突​

在多分支并行处理时，每一个分支可能是基于不同版本的主干分支创建的。如果每个分支都独立运行而不进行合并，就没有问题，但是如果在后续操作过程中进行合并的话，就有可能产生冲突。比如b1, b2的两个分支都是基于master分支创建出来的。B1分支如果和B2分支修改了同一份文件的话，那么在合并时，以哪一个文件为准呢，这就是所谓的冲突。 这时需要我们手动的处理冲突了，判断以哪个文件为主。

Git远程仓库(remote)​

此前我们对于git的所有操作均是在本地进行的，相当于自己一个人玩单机游戏。但是在开发过程中，我们是多人进行协作开发写项目的。这时我们需要一个远程的git仓库，远程的git仓库与本地的git仓库没有什么区别，不同之处在于远程的git仓库可以被多人同时访问使用，方便我们协同开发。 在实际工作中，git的服务器通常由公司搭建内部使用或是购买一些公共的私有git服务器。我们学习阶段，直接使用一些开放的公共git仓库，目前常用的库有两个：GitHub和Gitee(码云)

  1. GitHub手动创建仓库

  2. 将本地仓库上传到git

git remote add origin https://github.com/crtdya/xuyao.git  # git remote add <remote name>   <url>   此时git本地仓库已经与远程仓库关联上了
git branch -M main # 修改分支名字为main
git push -u origin main # 将代码上传到服务器上

注意：推送代码之前，一定要先从远程库中拉取最新的代码

gitignore​

默认情况下，git会监视项目中的所有内容，但是有一些内容，比如node_modules目录中的内容，我们不希望它被git所管理。我们可以在项目目录中添加.gitignore文件来设置那些需要git忽略的文件。
